# STATUS 날짜 선택 표시 전환 설계

2026-07-06 승인됨.

## 목적

날짜 셀 클릭 시 보기 다이얼로그 대신 STATUS 패널 하단(진행률 바 아래)에
선택 날짜의 정보를 표시한다. 수정은 [수정] 버튼으로 기존 다이얼로그를
수정 모드로 바로 연다.

## 표시 규칙 (진행률 바 하단)

- **과거**: 출근 시간 / 퇴근 시간 / 계획 시간: (가)계획 HH:MM~HH:MM /
  상태 / 메모. 상태는 기존 로직 재사용: 미출근(회), 조기 퇴근(주황),
  정상 퇴근(녹), 퇴근 완료(회·판정 불가), 퇴근 미기록(노랑 bold ⚠).
  조기 판정은 그날 출근+계획(휴가 차감·구간 연장 포함) 역산 예상 퇴근과
  실제 퇴근 비교.
- **오늘**: 기존과 동일 (출근/퇴근 예정/계획 퇴근/체류/남은/상태/메모).
- **미래**: 실 계획: Xh Ym / (가)계획 시간: HH:MM ~ HH:MM / 메모.
  상태 라인 없음.

## 버튼

- 오늘(미퇴근): [수정] [퇴근] / 오늘(퇴근 완료): [수정] [퇴근 취소]
  ('재퇴근' 제거 — 취소 후 다시 퇴근으로 대체)
- 과거·미래: [수정] [오늘] — '오늘' 클릭 시 선택=오늘, 캘린더=현재 월
- [수정]: 기존 다이얼로그를 곧바로 수정 모드로 연다

## 캘린더

- 셀 클릭 = 선택 (다이얼로그 없음). 선택 셀은 오늘 셀과 다른 색 테두리로
  강조 (theme.BORDER_SELECTED).

## 구성

- `core/day_detail.py`(신규): `DayDetail`(kind past/today/future, 출퇴근 HH:MM,
  (가)계획 범위, 실 계획 분, 메모, 조기 퇴근 여부, 기록 유무),
  `build_day_detail(storage, plan_service, holidays, date, today)`. TDD.
- `core/stats.py`: 예상 퇴근 역산(clock_in+계획-휴가, 구간 연장)을 공용
  함수로 추출해 day_detail 과 공유.
- `ui/status_panel.py`: 과거/미래 라인 포맷 순수 함수 + 과거 상태 표시,
  선택 날짜 렌더링, 버튼 구성(on_edit/on_today 콜백 추가).
- `ui/calendar_widget.py`: 선택 셀 테두리 강조 (render_grid 에 선택 날짜 전달).
- `ui/day_dialog.py`: `start_in_edit` 파라미터 (수정 모드로 즉시 오픈).
- `ui/app.py`: `_selected_date` 상태 관리, 셀 클릭=선택, [오늘]=오늘·현재 월
  복귀, [수정]=다이얼로그 수정 모드 오픈.
