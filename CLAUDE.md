# work-widget

macOS용 근무시간 트래커 (PySide6/Qt). 전체화면 캘린더 모드와 소형 위젯 모드를
지원하며, 출퇴근 기록·계획 관리·월 현황 집계를 제공한다.

## 실행·테스트

```bash
./venv/bin/python main.py      # 앱 실행
./venv/bin/pytest -q           # 전체 테스트
```

- 시스템 python 이 아니라 반드시 `venv/` 를 사용한다.
- GUI 렌더링은 헤드리스 환경에서 자동 검증이 불가하다. UI 변경 시 로직·임포트
  검증까지만 수행하고, 시각적 확인은 사용자에게 요청한다.

## 구조

```
core/    # 순수 로직 (Qt 비의존) — 반드시 TDD 로 작성
  worktime.py        # 휴게 임계-정지 모델 (compute_work_seconds ↔ raw_seconds_for_net 역함수 관계)
  plan.py            # 날짜별 계획(분) 오버라이드, 요일 일괄 설정
  recognition.py     # 인정 범위 도메인. UI 표기는 '(가)계획'
  calendar_model.py  # DayCell 그리드, 법정 기준(주40h)·최대 가능(주52h) 시간
  stats.py           # 월 집계, 예상 퇴근, 진행률 4단계(progress_state)
  storage.py         # SQLite (attendance/plan/recognition 테이블)
ui/      # Qt 위젯. 색상·크기는 theme.py 디자인 토큰으로만 정의 (매직 넘버 금지)
tests/   # pytest. core 로직은 경계값까지 검증
```

## 도메인 규칙

- **휴게(임계-정지)**: 누적 4h→30분, 8h→30분, 12h→30분 정지. 예상 퇴근은
  `raw_seconds_for_net` 으로 역산 (계획 ≤4h 휴게 없음, <8h +30m, <12h +60m, ≥12h +90m).
- **(가)계획(인정 범위)**: 사전 정의된 출근 인정 시각 구간. 실 근로가 벗어나면
  경고 표시만 하고 work_seconds 계산은 바꾸지 않는다. 저장 시 범위 폭이
  실 계획을 채우는 데 필요한 체류시간(휴게 포함)보다 좁으면 차단.
- **법정 기준** = floor(말일/7×40) − 평일 공휴일×8h. **최대 가능** = floor(말일/7×52),
  공휴일 차감 없음.
- **진행률 바**: 법정 이내(녹, max=법정) → 초과~+20h(노랑, max=최대 가능) →
  +20h 초과(주황) → 최대 초과(빨강). 얇은 무텍스트 스타일 유지.

## 단위작업 종료 절차

모든 단위작업이 끝나면 순서대로 자동 수행한다 (질의 없이):

1. `./venv/bin/pytest -q` 전체 테스트 통과 확인
2. 커밋 (아래 커밋 규칙)
3. `./run.sh stop && ./run.sh` 로 위젯 재시작 — 변경사항이 실행 중인 앱에
   반영되도록 재기동한다

## 커밋 규칙

- **모든 단위작업이 끝나면 사용자에게 질의 없이 자동으로 커밋한다.**
  (전역 규칙의 "커밋 여부 질의" 를 이 프로젝트에서는 적용하지 않음)
- 한 커밋 = 한 논리 단위. 여러 기능이 같은 파일에 섞이면 중간 상태를 재구성해
  나누고, 각 커밋 스냅샷이 단독으로 테스트를 통과하는지 확인한다.
- 커밋 메시지는 한글, 전역 prefix 이모지 규칙(✨🐛🔧📝♻️🔥, UI 스타일은 💄) 사용.
